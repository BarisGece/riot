Files

RIOT can import/export files in these formats:

  • Delimited (CSV)

  • Fixed-Width

  • JSON

Files can be GZIP compressed, and local or remote:

  • /myfolder/myfile.csv

  • /myfolder/myfile.json.gz

  • https://example.com/path/dataset.csv

  • s3://mybucket/myfolder/myfile.json

For AWS S3 you can specify access and secret keys as well as the region for the bucket.

Delimited Files

CSV → Redis
$ riot file-import --file https://git.io/fjxPs --header --keyspace beer --keys id
Redis → CSV
$ riot file-export --scan-match beer:* --key-regex "beer:(?<id>.*)" --file /tmp/beers.csv --fields id name brewery_id abv --header --delimiter '|'
CSV → RediSearch

  1. Create an index with redis-cli

    FT.CREATE beers SCHEMA abv NUMERIC SORTABLE id TAG name TEXT PHONETIC dm:en style_name TEXT cat_name TEXT brewery_id TAG

  2. Import data into the index

    $ riot file-import --file https://git.io/fjxPs --header --include 1 3 4 5 6 --command ftadd --index beers --keys id

  3. Search for beers

    > FT.SEARCH beers "@abv:[7 9]"

JSON

RIOT can read and write JSON data in this format: [ { … }, { … } ]

JSON → Redis
$ riot file-import --file https://git.io/fjxPZ --filetype json --keyspace beer --keys id
Redis → JSON
$ riot file-export --file /tmp/beers.json --scan-match beer:* --key-regex "beer:(?<id>.*)" --fields id name brewery_id abv
Redis → Compressed JSON
$ riot file-export --file /tmp/beers.json.gz --scan-match beer:* --fields id name brewery_id abv

Databases

RIOT can connect to SQL databases through JDBC. Install the appropriate JDBC driver under the lib directory and modify the RIOT CLASSPATH accordingly:

  • *nix: bin/riot → CLASSPATH=$APP_HOME/lib/myjdbc.jar:$APP_HOME/lib/…

  • Windows: bin\riot.bat → set CLASSPATH=%APP_HOME%\lib\myjdbc.jar;%APP_HOME%\lib\…

For reference here are links to common database systems and related JDBC driver documentation:

RDBMS URL Format

Oracle

jdbc:oracle:thin:@myhost:1521:orcl

MS SQL Server

jdbc:sqlserver://[serverName[\instanceName][:portNumber]][;property=value[;property=value]]

MySQL

jdbc:mysql://[host]:[port][/database][?properties]

PostgreSQL

jdbc:postgresql://host:port/database

IBM Db2

jdbc:db2://host:port/database

SQLite

jdbc:sqlite:sqlite_database_file_path

SQL → Redis

$ riot db-import --url jdbc:hsqldb:mem:mymemdb --sql "SELECT * FROM beers" --keyspace dbbeer --keys id

Redis → SQL

This command exports all hashes in keyspace beer:<id> to the database:

$ riot db-export --scan-match "beer:*" --key-regex "beer:(?<id>.*)" --url jdbc:hsqldb:mem:mymemdb --sql "INSERT INTO beers (id, name) VALUES (:id, :name)"

Generators

RIOT can be used to create random data in Redis with 2 different generators.

Simple

The simple generator produces fixed-length fields with 2 fields by default:

  • index: monotonous integer sequence

  • partition: index of the partition (thread) generating the data, e.g. if you have 8 threads generating data each will have a different partition index between 0 and 7.

You can also configure it to generate fixed-sized fields with --field <name=size> options.

For example the following command generates hashes in the keyspace test:<index> with fields field1 and field2 of respectively 100 and 1,000 bytes:

$ riot gen -d field1=100 field2=1000 --batch 100 --threads 3 --max 10000 --keyspace test --keys index

Faker

This data generator relies on the Faker library. The supported data types are described here.

Example #1: People
$ riot gen --faker id=index firstName=name.firstName lastName=name.lastName address=address.fullAddress --max 100 --keyspace person --keys id
> HGETALL person:1
1) "address"
2) "036 Robbin Points, North Sonia, PA 42251"
…
Example #2: Game of Thrones
$ riot gen --faker name=gameOfThrones.character --max 10000 --command sadd --members name --keyspace got:characters
> SMEMBERS got:characters
1) "Nymella Toland"
2) "Ysilla Royce"
…
RediSearch Index Introspection

RIOT can introspect an existing RediSearch index (specified with the --faker-index option) and generate Faker data accordingly:

$ riot gen --faker-index beerIntrospection --max 100 --command ftadd --index beerIntrospection --keys id

Redis

The Redis connector allows for replicating data between two Redis databases using scan and optionally keyspace notifications (--listener).

$ riot -s localhost:16379 replicate -s localhost:16380 --listener

The source database is specified first (localhost:16379), and the target database is after the replicate command (localhost:16380)

Processors

RIOT can process records as they are being read. There are 2 processor approaches: field expressions or script processor.

Field expressions

You can specify field expressions to process key/value pairs using the Spring Expression Language (SpEL): field1=<exp>, field2=<exp>, …​

The input record is accessed through its field names (e.g. field3=field1+field2).

The processor also exposes the following variables that can be called with the # prefix:

  • redis: Redis connection to issue any command, e.g. name=#redis.hgetall('person1').lastName

  • date: date parser/formatter, e.g. epoch=#date.parse(mydate).getTime()

  • context: exposes processor context, e.g. id=#context.index

$ riot file-import --file "https://data.lacity.org/api/views/rx9t-fp7k/rows.csv?accessType=DOWNLOAD" --header --spel-date-format "MM/dd/yyyy HH:mm:ss a" --spel "EventStartDate=remove('Event Start Date')" "EpochStart=#date.parse(EventStartDate).getTime()" "index=#context.index()" --keyspace event --keys Id

Script evaluator

Use an external or inline script to provide the processor logic.

Javascript example
function process(item) {
    item.name = item.name.toUpperCase();
    return item;
}
process(item);
$ riot gen --faker id=index firstName=name.firstName lastName=name.lastName address=address.fullAddress --script "function process(item) { item.address = item.address.toUpperCase(); return item; } process(item);" --max 100 --keyspace person --keys id
